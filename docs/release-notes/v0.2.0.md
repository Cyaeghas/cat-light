# cat-light v0.2.0

Reliability and parser hardening release.

## Highlights

- Hardened Codex and Claude local JSONL parsing for provider schema drift.
- Added fixtures for approval flow, failed tools, patch edits, hook errors, nested Claude tool-result errors, schema drift, and same-name sessions.
- Improved token/context extraction for prompt/completion aliases, cache read/write fields, reasoning tokens, context aliases, and wrapper/API events.
- Prevented passive log sessions from merging when different projects contain same-name JSONL files.
- Added `doctor` checks for local log roots, stale or malformed JSONL, hook events, SQLite backend availability, and Windows floating monitor startup registration.
- Pinned the GitHub Actions Windows runner to `windows-2025` for more predictable release builds.

## Verification

- MSVC Release build with bundled SQLite.
- JSONL fixture tests.
- SQLite fixture tests.
- Local HTTP smoke test for `/health` and `/state`.

## Notes

Claude quota support remains conservative because Claude Code local credential and quota endpoint behavior can change. This release focuses on local-first reliability: session state, token/context extraction, hooks, doctor checks, and repeatable fixtures.
